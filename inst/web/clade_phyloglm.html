Influential clade detection - Phylogenetic Logistic Regression

Usage

clade_phyloglm(formula, data, phy, btol = 50, track = TRUE, clade.col, n.species = 5, ...)

Arguments

formula
The model formula
data
Data frame containing species traits with row names matching tips in phy.
phy
A phylogeny (class 'phylo') matching data.
btol
Bound on searching space. For details see phyloglm
track
Print a report tracking function progress (default = TRUE)
clade.col
The name of a column in the provided data frame with clades specification (a character vector with clade names).
n.species
Minimum number of species in the clade in order to include this clade in the leave-one-out deletion analyis. Default is 5.
...
Further arguments to be passed to phyloglm

Value

The function clade_phyloglm returns a list with the following components:

formula: The formula

full.model.estimates: Coefficients, aic and the optimised value of the phylogenetic parameter (e.g. alpha) for the full model without deleted species.

clade.model.estimates: A data frame with all simulation estimates. Each row represents a deleted clade. Reported are the calculated regression intercept (intercept), difference between simulation intercept and full model intercept (DFintercept), the percentage change in intercept compared to the full model (intercept.perc) and intercept p-value (pval.intercept). All of these are also reported for the regression slope (DFslope etc.). Additonally, model aic value (AIC) and the optimised value (optpar) of the phylogenetic parameter are reported.

data: Original full dataset.

errors: Clades where deletion resulted in errors.

Description

Estimate the impact on model estimates phylogenetic Logist regression after removing clades from the analysis.

Details

Currently only logistic regression using the "logistic_MPLE"-method from phyloglm is implemented.

clade_phyloglm detects influential clades based on difference in intercept and/or slope when removing a given clade compared to the full model including all species.

Currently, this function can only implement simple linear models (i.e. $y = a + bx$). In the future we will implement more complex models.

Output can be visualised using clade_plot.
